On the API VM (Linux):


openssl rand -base64 48
# example: w8K2mP9nF3qY7vX1jL5tR0aZ4bU6cE8dN2sV7hG3iH9kJ5oP1mQ4rT8yU2wB6xC4
On Windows (PowerShell):


[Convert]::ToBase64String((1..48 | ForEach-Object { Get-Random -Maximum 256 } | ForEach-Object { [byte]$_ }))
Or the cleaner version using .NET's RNG:


$bytes = New-Object byte[] 48
[Security.Cryptography.RandomNumberGenerator]::Create().GetBytes($bytes)
[Convert]::ToBase64String($bytes)
Anywhere with Node:


node -e "console.log(require('crypto').randomBytes(48).toString('base64'))"
Generate three separate values — one each for JWT_ACCESS_SECRET, JWT_MFA_SECRET, and MFA_ENCRYPTION_KEY. Don't reuse the same value. The MFA_ENCRYPTION_KEY specifically must decode to exactly 32 bytes — openssl rand -base64 32 gives you that (44 base64 chars).

Add them to /opt/talvex/app/.env.prod:


JWT_ACCESS_SECRET=<paste output 1>
JWT_MFA_SECRET=<paste output 2>
MFA_ENCRYPTION_KEY=<paste output 3>
Important: once these go into prod, save them in your password manager / Key Vault. If MFA_ENCRYPTION_KEY is ever lost, every encrypted MFA secret becomes unrecoverable and those users will need to re-enrol MFA.
