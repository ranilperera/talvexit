Bypass-flag note

Why no env changes were made. The instruction asked to bypass KYC, ABR, Stripe and OTP for the run, then re-enable after. These flags only affect runtime API requests (when a real HTTP client hits the server). Vitest tests mock those services at the module level, so the flags would have zero impact on this run.

No .env.local was edited. Nothing needs to be reverted. The server you launch tomorrow with pnpm dev behaves exactly as before this run. If you need a runtime-driven E2E (real HTTP, real DB writes), that requires a different setup — see Recommendations.
