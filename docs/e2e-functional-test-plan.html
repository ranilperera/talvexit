TalvexIT — End-to-End Functional Test Plan v1.0

Manual QA script — covers the full marketplace lifecycle from account creation through order completion.
Date: 2026-05-08  ·  Owner: QA  ·  Scope: Web UI + API contracts  ·  Environments: local dev (http://localhost:3000) → staging
How to use this document. Run each phase in order — later phases depend on data created in earlier phases. Mark each test case PASS, FAIL, or BLOCKED in the rightmost column. Capture screenshots into /qa-evidence/<TC-ID>.png on every FAIL. The 12-combo tax matrix is exercised three times (orders, proposals, invoices) — discrepancies between phases indicate a regression in decideGstTreatment() or one of its persistence sites.

Phases

  1. Test data & environment setup

  3. Onboarding & verification
